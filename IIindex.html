Nuvem (APIs Externas)

Local (Soberania Própria)

  • Soberania absoluta dos dados corporativos (sem tráfego de internet).
  • Custo totalmente previsível (apenas infraestrutura e energia fixa).
  • Latência ultrabaixa via rede de dados interna (LAN).
  • Manutenção, controle e atualizações sob responsabilidade da TI interna.

Motor de Inferência

Por que escolher o vLLM?

Gerenciar 20 desenvolvedores simultâneos em uma única GPU exige otimização profunda de VRAM.

O vLLM utiliza PagedAttention para alocar memória de chaves/valores de forma dinâmica, evitando a fragmentação física. Isso reduz drasticamente o desperdício de VRAM em relação a motores simples (como Ollama ou LM Studio), permitindo paralelizar requisições.

Fluxograma

Lógica de Decisão do Clima

O script opera em loop contínuo (leituras de 10 a 30s para resguardar a integridade física dos sensores e relés):

[Início] ➔ [Leitura Sensor] ➔ [Análise Limites] ➔ [Decisão Atuação] ➔ [Banco & Logs] ↺ Repete

Estados Climáticos de Atuação

  • Abaixo de 30°C: Sistema em standby. Coolers desligados ou na velocidade mínima (economia de energia).
  • Entre 30°C e 40°C: O Pi aciona o Módulo Relé 1, ativando os exaustores em potência regular.
  • Acima de 40°C (Emergência): Aciona o Relé 2 de exaustão secundária/emergência em rotação máxima e dispara alerta imediato de TI.
  • Banco de dados: Cada ciclo insere dados históricos contendo: [Data/Hora, Temp, Umid, Status_Cooler].
Linguagem Escolhida

Por que usar Python?

Python é a linguagem ideal para esta infraestrutura IoT devido a:

  • Suporte Nativo: Bibliotecas altamente maduras para controle direto de pinos (RPi.GPIO ou gpiozero).
  • DHT22 Estável: Acesso direto aos dados do sensor via biblioteca da Adafruit_DHT.
  • Armazenamento Leve: Manipulação de banco de dados nativo (sqlite3) com pouquíssimas linhas de código.

Visão do Código de Controle

import RPi.GPIO as GPIO
import Adafruit_DHT
import sqlite3
# Configurações do Sensor e Pinos

def analisar_clima(temp, umid):
    if temp > 40:
        GPIO.output(rele_1, True)
        GPIO.output(rele_2, True) # Potência Máxima!
        disparar_alerta_telegram(temp)

        GPIO.output(rele_1, False)
